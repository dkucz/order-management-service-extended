# Release v1.6.0

## Date
2026-03-15

## Highlights
- `order_stream` is now the canonical exchange for downstream integrations.
- OpenTelemetry context propagation is enabled for RabbitMQ publish/consume flows.
- Audit logging continues to be best-effort, but correlation ID propagation was tightened.

## Upgrade Notes
- Consumers using `order_events` should migrate immediately.
- If your monitoring depended only on `x-correlation-id`, keep it; tracing augments it rather than replacing it.

## Known Gaps
- Internal DLQ setup still uses legacy hyphenated constants in some code paths.
- Public pagination docs and swagger comments are not fully reconciled yet.
